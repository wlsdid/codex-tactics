# Study Note - Manual Validation and Capture

## Concept

Automated checks are useful, but a game portfolio also needs player-facing evidence. A screenshot or GIF shows that the feature works in the actual Unity scene and communicates the result faster than code alone.

## Applied checklist idea

A good manual validation checklist should include:

1. How to rebuild the scene
2. Which validation menu to run
3. What visible state to check during play
4. What screenshots or GIFs to capture
5. What wording is safe before Unity Play Mode is actually tested

## Portfolio lesson

For a portfolio, proof is part of the work. A feature is easier to explain when the repository contains:

- code
- test/validation steps
- devlog
- study note
- screenshot or GIF plan

## Important wording

Until Unity has actually been run, use:

> Static checks passed outside Unity. Unity manual validation is pending.

Do not claim:

> Unity Play Mode passed.

unless it was really tested in Unity.
