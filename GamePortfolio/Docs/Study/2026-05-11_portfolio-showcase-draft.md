# Study Note - Portfolio Showcase Draft

## Concept

A game portfolio is not only the final game. It should also explain the developer's decisions, structure, tests, and improvement process.

## Applied in this project

`Docs/PortfolioShowcaseDraft.md` organizes the current prototype into portfolio-friendly sections:

- project overview
- playable loop
- battle state flow
- data-driven values
- tactical player choices
- enemy intent
- result summary
- editor validation
- screenshot/GIF targets

## Why this matters

If the project is only shown as code, reviewers may miss the design thinking. A showcase draft turns the implementation into a story:

```text
Problem: player needs readable tactical choices.
Solution: AP skill, Guard, enemy intent, and result feedback.
Evidence: Unity screenshots/GIFs and validation logs.
```

This makes later README/Notion polishing easier.
