Codex Tactics — Architecture

Unity 2D Turn-Based RPG Prototype — Component Relationships & Data Flow

Battle Flow

  • • 4-state machine: Start → PlayerTurn → EnemyTurn → Victory/Defeat
  • • Coroutine-driven EnemyTurnRoutine (1s delays for readability)
  • • OnClick* handlers triggered by uGUI buttons
  • • Config* helpers fall back to defaults if no ScriptableObject assigned

Data-Driven Design

  • • BattleBalanceConfig: 1 ScriptableObject = all tuning values
  • • StageData/EnemyData: new encounter = new asset, no code
  • • EnemyPatternData: AI behavior per enemy type
  • • SkillData: skill parameters (power, AP cost, element, status)

Automation & Docs

  • • Editor auto-builders recreate test scene from code
  • • Auto-test runner validates battle logic through all states
  • • Per-feature Devlog + Study Note pair
  • • Balance table, manual checklist, showcase draft
