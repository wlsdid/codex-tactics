# Devlog - Manual Unity Validation Checklist

## Goal

Create a copyable checklist that the user can run inside Unity and paste into a devlog or Notion page as proof of validation.

## What changed

- Added `Docs/ManualUnityValidationChecklist.md`.
- Split the checklist into practical sections:
  - setup
  - editor menu validation
  - Play Mode start state
  - Fire Skill / Burn
  - Guard
  - Enemy Intent / Heavy Slam
  - result summary
  - Retry reset
  - capture readiness
- Added a small validation record block for `PASS / FAIL` notes.
- Linked the checklist from README.

## Why it helps

The existing capture guide explains what to record. This checklist is more direct: it gives the user a step-by-step PASS/FAIL sheet for Unity manual testing.

That makes the portfolio stronger because the project can show both automated editor-menu validation and a human Play Mode verification record.

## Manual follow-up

Unity still needs to be opened by the user. After completing the checklist, update the devlog with the copied validation record.
