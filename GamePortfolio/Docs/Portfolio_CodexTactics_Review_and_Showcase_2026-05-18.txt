============================================================
1. 한 줄 소개
============================================================
Codex Tactics는 Unity 2D로 제작한 턴제 RPG 전투 포트폴리오입니다. 타이틀 화면, 스테이지 선택, 전투, 결과 화면, 저장/진행도, 스킬/속성/상태이상/브레이크/스테이지 기믹/자동 전투/검증 자동화까지 포함한 플레이 가능한 수직 슬라이스(vertical slice)입니다.

포트폴리오에서 강조할 핵심은 단순 전투 구현이 아니라, 데이터 기반 전투 설계와 자동 검증, 단계별 UI/UX 개선, 스테이지별 전장 기믹을 실제 전투 로직에 연결한 점입니다.

============================================================
2. 현재 플레이 흐름
============================================================
Title Scene
→ Stage Select Scene
→ Battle Scene
→ Result / Continue / Retry / Stage Select
→ 다음 encounter 또는 다음 stage unlock

현재 구성:
- 6개 스테이지
- 각 스테이지 2개 encounter: 일반전 + 보스전
- 총 12개 전투 encounter
- 스테이지 클리어 시 다음 스테이지 및 스킬 진행도 해금
- JSON 저장을 통한 진행도 유지

3. 주요 게임 시스템
============================================================

3-1. 턴제 전투 상태 머신
- 핵심 파일: Assets/Scripts/Battle/BattleManager.cs
- 상태: Start / PlayerTurn / EnemyTurn / Victory / Defeat
- 플레이어 행동:
  - 기본 공격 Slash
  - Fire Bolt
  - Ice Lance
  - Lightning Strike
  - Earth Wall
  - Guard
  - Item 사용
  - End Turn
- 적 행동:
  - 일반 공격
  - 주기적 강공격
  - Stun 시 턴 스킵
  - HP 30% 이하 Enrage

포트폴리오 설명 포인트:
- 플레이어와 적 행동을 턴 단위로 분리하여 전투 흐름을 명확히 구성했다.
- 결과 화면과 진행도 저장까지 이어지는 완성된 전투 루프를 구현했다.

3-2. AP 기반 스킬 시스템
- AP 최대치: 3
- 최신 밸런스: 플레이어 턴 시작 시 AP 2 회복
- 스킬별 AP 비용:
  - Slash: 0 AP
  - Ice Lance: 1 AP
  - Fire Bolt: 2 AP
  - Earth Wall: 2 AP
  - Lightning Strike: 3 AP

포트폴리오 설명 포인트:
- 무료 기본 공격과 고비용 고효율 스킬을 나누어 선택지를 만들었다.
- Void Drain 같은 스테이지 기믹이 AP 자원 관리와 직접 연결된다.

3-3. 속성/약점 시스템
- 주요 파일: SkillData.cs, EnemyData.cs, BattleManager.cs
- 원소: Physical, Fire, Ice, Lightning, Nature, Earth, Dark, Light
- 약점 공격 시 데미지 증가 및 Break gauge 감소
- UI에 적 원소/약점 정보와 impact text 표시

포트폴리오 설명 포인트:
- 단순 공격 반복이 아니라, 적 약점을 보고 스킬을 선택하는 전투 판단 구조를 만들었다.

3-4. 상태이상 시스템
- Burn: 턴마다 지속 피해. 최신 밸런스 기준 5 피해.
- Stun: 적 턴 스킵.
- Shield: Earth Wall 사용 시 보호막 20.
- Guard: 다음 적 공격 피해 감소.
- Break: 약점 공격으로 gauge를 깎고, Break 상태에서 추가 피해 보너스.

포트폴리오 설명 포인트:
- 상태이상이 단순 텍스트가 아니라 실제 전투 흐름, 데미지, UI 피드백에 영향을 준다.

3-5. 스테이지별 전장 기믹
핵심 파일:
- Assets/Scripts/Data/StageData.cs
- Assets/Scripts/Battle/BattleManager.cs
- Assets/Editor/BattleAutoTestRunner.cs

현재 Stage Modifier 구성:
1. Stage 1 — Tutorial Field
   - 안전한 튜토리얼 전장.
   - 특별한 위험 없이 기본 전투 학습.

2. Stage 2 — Pack Pressure
   - 적 강공격 주기 단축.
   - strong attack every 3 → every 2.
   - 검증: skill help와 battle log에 반영.

3. Stage 3 — Stoneguard
   - 적 Break gauge 증가.
   - Break: 2/2 → 3/3.
   - 검증: Break UI와 BattleAutoTestRunner에서 확인.

4. Stage 4 — Storm Surge
   - 3번째 enemy turn마다 전장 번개 피해.
   - 플레이어에게 고정 8 hazard damage.
   - battle log, message, impact text, totalDamageTaken에 반영.

5. Stage 5 — Void Drain
   - 2번째 enemy turn마다 AP 압박.
   - AP가 있으면 AP 1 감소.
   - AP가 0이면 HP 5 hazard damage.
   - AP 0 분기까지 자동 테스트 보강 완료.

6. Stage 6 — Radiant Trial
   - 최종 시험형 복합 기믹.
   - 강공격 주기 단축 + Break gauge 증가.
   - Stage 6 battle log, skill help, Break UI에서 확인 가능.

포트폴리오 설명 포인트:
- StageData의 modifier metadata를 Stage Select UI와 BattleManager가 함께 사용한다.
- “설명만 있는 기믹”이 아니라 실제 전투 룰과 자동 테스트에 연결된 시스템이다.

3-6. 아이템 시스템
최신 구성:
- Potion: HP 30 회복
- Hi-Potion: HP 60 회복
- Ether: AP 2 회복
- Full Ether: AP 전체 회복 성격(effectValue 99로 max AP까지 회복)

기능:
- 아이템 수량 관리
- 사용 시 HP/AP 반영
- 회복 popup, battle log, SFX 연결
- Auto Battle AI가 상황에 따라 Potion/Ether 사용

포트폴리오 설명 포인트:
- 전투 중 자원 회복 선택지를 추가해 전략성을 확장했다.

3-7. Auto Battle AI
- Guard / Item / Weakness / Lightning / Ice / Earth / Fire / Basic 우선순위 기반.
- HP/AP 상황에 따라 아이템 사용.
- 적 약점과 보유 AP를 고려해 스킬 선택.

포트폴리오 설명 포인트:
- 간단한 규칙 기반 AI로도 전투 시스템 테스트와 시연 편의성을 높였다.

3-8. 결과/보상 시스템
핵심 파일:

표시 요소:

6-1. Critical

- working tree는 깨끗하고 origin/main과 동기화되어 있다.

6-2. Warnings / 추후 개선 권장
1) BattleUI의 overlay pulse coroutine 중복 가능성
   - Guard/Burn/Stun overlay가 UpdateAllUI 호출 중 반복적으로 StartCoroutine될 수 있다.
   - 즉시 테스트 실패는 없지만, 장시간 플레이 시 중복 coroutine이 쌓일 수 있다.
   - 개선안: overlay별 Coroutine handle을 저장하고, 이미 실행 중이면 새로 시작하지 않도록 방지.

2) SkillProjectile의 FindObjectOfType 사용
   - hit spark 생성 시 Canvas를 FindObjectOfType으로 찾는다.
   - 현재 규모에서는 문제 없지만, UI가 커지면 비용/불확실성이 생길 수 있다.
   - 개선안: projectile parent canvas를 Spawn 시 전달하거나 cached canvas를 사용.

3) README가 최신 배치 상태와 다름
   - README에는 “37개 배치”와 4 stage 기준 내용이 남아 있다.
   - 현재는 Batch 55까지 진행되었고 6 stage / 12 encounters / Stage 2~6 modifier가 구현되어 있다.
   - 개선안: README를 최신 상태로 갱신.

4) Batch 번호 중복
   - git log에 Batch 49/50/51 명칭이 일부 중복되어 있다.
   - 기능에는 영향 없지만, 포트폴리오 기록용으로는 “기능 단위” 타임라인으로 재정리하는 것이 좋다.

6-3. Looks Good
- StageData가 스테이지/적/기믹 데이터의 원천 역할을 하도록 정리되어 있다.

7. 포트폴리오용 기술 어필 문장
============================================================

짧은 소개:
“Codex Tactics는 Unity 2D 기반 턴제 RPG 전투 수직 슬라이스입니다. 6개 스테이지, 12개 encounter, AP 기반 스킬, 속성 약점, 상태이상, Break, 아이템, 자동 전투, 결과 보상, 진행도 저장, 스테이지별 전장 기믹을 구현했습니다. 특히 StageData 기반 데이터 구조와 Unity Editor 자동 검증 도구를 통해 기능 추가 시 UI/전투 로직 정합성을 확인할 수 있도록 만들었습니다.”

기술 중심 소개:
“전투 로직은 BattleManager의 상태 머신으로 관리하고, UI는 BattleUI, 결과 계산은 BattleResultEvaluator/Presenter, 스테이지/적 정보는 StageData/EnemyData로 분리했습니다. 이 구조를 통해 전투 규칙, 표시, 데이터 정의가 서로 독립적으로 확장될 수 있게 설계했습니다.”

문제 해결 중심 소개:
“초기에는 스테이지 선택 화면의 표시 데이터와 실제 전투 보상/기믹 데이터가 어긋날 위험이 있었습니다. 이를 해결하기 위해 StageData를 단일 데이터 원천으로 사용하도록 개선했고, Stage Select와 BattleManager 양쪽에서 같은 modifier metadata를 참조하게 만들었습니다. 이후 자동 테스트에서 UI 표시와 전투 실제 적용을 함께 검증하도록 보강했습니다.”

검증 중심 소개:
“Unity Editor 메뉴 기반 자동 테스트와 batchmode 검증을 만들어, 기능 추가 후 Battle logic, scene wiring, game flow를 반복 검증했습니다. 단순히 플레이해 보는 수준이 아니라 Stage 2~6 기믹의 실제 수치 변화와 UI 로그까지 자동으로 확인하도록 테스트를 확장했습니다.”
