# Day 1 - 프로젝트 방향 설정

## 오늘 한 일

- 포트폴리오 목적을 “대학교 과진입용 게임 포트폴리오”로 정리했다.
- 장르는 2D 픽셀 턴제 RPG로 결정했다.
- 처음부터 큰 RPG를 만들지 않고, 아군 1명 vs 적 1명의 전투 프로토타입부터 완성하기로 했다.
- GitHub에 올릴 수 있도록 프로젝트 폴더와 README, 문서 구조를 만들었다.
- Unity에서 사용할 C# 전투 스크립트 초안을 작성했다.

## 왜 턴제 RPG를 선택했는가

턴제 RPG는 캐릭터 데이터, 전투 흐름, 스킬, UI, 밸런싱 과정을 보여주기 좋다. C#의 class, enum, 함수, 리스트 같은 개념을 실제 게임 기능에 연결해서 공부하기에도 적합하다.

## 막힌 점

- C#과 Unity 경험이 부족하다.
- 전체 게임 범위를 크게 잡으면 완성하지 못할 가능성이 있다.
- GitHub 원격 저장소 생성은 현재 토큰 권한 문제로 자동 생성이 막혔다.

## 해결 방향

- 먼저 작은 전투 하나를 완성한다.
- 공부 내용은 “왜 배웠고 어디에 적용했는지” 중심으로 기록한다.
- GitHub 저장소는 직접 만들거나 토큰 권한을 수정한 뒤 연결한다.
