# Day 1 - 프로젝트 방향 설정

## 오늘 한 일

- 포트폴리오 목적 정리
- 게임 장르를 픽셀 턴제 RPG로 결정
- Unity + C# 사용 결정
- 스테이지 클리어형 구조로 범위 축소
- GitHub/Notion/README 구조 계획

## 왜 이 장르를 선택했는가

턴제 RPG는 캐릭터 데이터, 스킬, UI, 전투 흐름, 밸런싱을 모두 보여줄 수 있어 포트폴리오에 적합하다. 실시간 액션보다 구현 범위를 관리하기 쉽고, C#의 클래스와 객체지향 개념을 공부하면서 적용하기 좋다.

## 막힌 점

- C#과 Unity 경험이 부족하다.
- 만들고 싶은 기능은 많지만 처음부터 크게 만들면 완성하기 어려울 수 있다.

## 해결 방향

먼저 아군 1명과 적 1명이 싸우는 작은 전투 프로토타입을 완성한다. 이후 AP, Break, 스킬, 스테이지를 하나씩 확장한다.

## 다음 목표

- Unity 프로젝트 생성

- Player/Enemy 임시 오브젝트 배치
